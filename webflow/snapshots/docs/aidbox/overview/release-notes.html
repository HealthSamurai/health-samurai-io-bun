Release Notes

Aidbox FHIR server release notes with latest features, improvements, and updates for edge, latest, and stable versions.

February 2026 edge

January 2026 latest, 2601

  • Aidbox FHIR server
    • Improved support for FHIR Topic-based subscriptions.
    • Loading FHIR packages from local filesystem is now supported.
    • Added support for [$current-canonical](../artifact-registry/artifact-registry-overview.md#current-canonical-operation ) operation.
    • Added AWS EventBridge as an AidboxTopicDestination. See full tutorial
    • Implemented AWS Default credentials authorization. See full tutorial
    • Performance improvemens:
      • +25% on CRUD operations
      • +80% on FHIR bundle import operations.
    • Optimized memory usage on server initialization.
    • Enhancements to HL7V2 incoming mapper.
    • Support for Aidbox behind HTTP(S) proxies. See full proxy example .
    • Added GraphQL Federation support. See full example .
    • Added GraphQL _filter support.
    • Add support for new OAuth scopes for graphql and ViewDefinition operations documentation
    • Improve SearchParameters UI in Resource Browser.
    • Improved FAR UI and performance on terminology operations.
    • Published official Aidbox grafana dashboard Dashboard
    • Deprecated the smartbox image, migration to Aidbox + FHIR App Portal is required.
  • Minor updates
    • 2512.1 - Optimize memory usage and improve performance.
    • 2512.2 - Fix FHIR validation issues.

December 2025 stable, 2512, LTS

  • Aidbox FHIR server
    • FHIR Topic-based Subscriptions are now supported.
    • Updated documentation for configuring Two Factor Authentication with the Aidbox security provider.
    • CRUD operation performance improved by 30% through internal optimizations.
    • Multiple bug fixes in the validator.
    • Security-related bug fixes.
  • Aidbox Forms
    • Supported the ability to create dynamic parameterized valueSets that filter answer options based on other form fields using FHIRPath expressions.
    • Created an open-source React renderer for custom form rendering, now available in alpha stage.
    • Supported $questionnaire-package operation for the Da Vinci DTR flow.
    • Implemented the tab-container widget to display groups as form tabs.
    • Redesigned the form outline with item search support.
    • Implemented all widgets for entryMode.
    • Supported theme usage for entryMode.
  • Minor updates
    • 2507.4 - Fixed vulnerabilities

November 2025 stable, 2511

  • Aidbox FHIR server
    • Improvements in Implementation Guide (IG) Management:
      • Added support for integration with external NPM package registries for IG loading.
      • Introduced pinning and tree shaking for optimizing IG dependency resolution.
      • Released the new FAR package management API for managing published IG artifacts.
      • Published a tutorial on loading FHIR IGs via Init Bundle.
      • Support multiple versions of the same IG in the same Aidbox instance.
    • Organization-based hierarchical access control improvements:
      • Added a tenant-aware login API for organization-based hierarchical access control.
      • Tenant-aware group-level export is now supported.
      • Added a tenant-aware AuditEvent generation.
    • Support for GCP Workload Identity for the $export in both GCP and Azure environments.
    • Support for Azure Workload Identity to access Azure Blob Storage.
    • Contained resources are now supported in GraphQL API.
    • Improved error handling on Bundle validation and processing.
  • Aidbox Forms
    • Added the ability to set up an autosave interval in SDCConfig.
    • Supported multitenancy in Form UI for creating and sharing forms between organizations.
    • Added support for the NHS design system to create forms according to the NHS style guide and embed them into the NHS app.
    • Improved PDF/Questionnaire import with clearer error display and easy navigation between errors.
  • Minor updates
    • 2510.1 - Fix redirects after registration email confirmation
