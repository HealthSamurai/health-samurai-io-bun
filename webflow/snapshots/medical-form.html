Solution Architecture

Flexible, Scalable, and Customizable

Option1: Using Aidbox FHIR server to store collected data

Aidbox Forms

  • UI Form Builder: A drag-and-drop interface for non-technical users, supporting complex logic and customization.
  • Form Renderer: Full customization for seamlessly integrating your app’s design and workflows.
  • FHIR SDC API: Collect and exchange data in a standardized FHIR format, following the FHIR Structured Data Capture (SDC) Implementation Guide.
  • Form Library:  Access nearly 3,000 ready-to-use, customizable forms.

Aidbox FHIR Server

Aidbox Forms is deployed with the Aidbox FHIR Server providing SDC Forms with the following functionality:

  • FHIR API & FHIR Storage: Store, retrieve, and interact with FHIR-compliant data.
  • Subscriptions: Get real-time updates by subscribing to FHIR resource changes.
  • Authentication (Auth): Secure access with a comprehensive authentication mechanism, ensuring only authorized users can interact with your forms and APIs.
  • Audit Logging: Track every action for security and compliance.
  • Analytics Module: Analyze FHIR data using flat resource representation.

* Aidbox Forms with the Aidbox FHIR Server is delivered as a Docker container, making deployment easy on any infrastructure.

‍

Aidbox Forms are provided with their own FHIR server, which is required to run the forms, store form templates, and can also be used for securely storing collected data.

Option2: Using your FHIR-Server for data storage

In this setup:

  • Pre-filled Forms: FHIR questionnaire will be pre-filled with data from your storage.
  • Form Submission: Once the form is submitted, the extracted data will be saved directly into your storage.
  • Form Creation and Storage: Forms will be created and stored in the Aidbox FHIR Server.
  • Audit Logging: All logging will be handled within the Aidbox FHIR Server for full traceability.

This flexibility ensures that Aidbox Forms can adapt to your infrastructure requirements while keeping your data secure and centralized.

If you need to store data in your storage, Aidbox Forms can be seamlessly integrated and used as a repository for creating and storing FHIR questionnaire templates. The data will remain in your storage, ensuring you retain full control.
